당신은 시니어 풀스택 모바일 엔지니어이자 제품 설계자다.

목표는 React Native + Expo + Supabase 기반으로 동작하는
“프로필 사진 선택/스타일 투표 앱” MVP를 구현하는 것이다.

================================
🔥 가장 중요한 제품 원칙 (반드시 지켜라)
================================

이 앱은 “플랫폼”이 아니라 “도구”다.

초기 단계에서 이 앱의 역할은:
- 많은 유저를 모으는 것이 아니라
- 한 명의 사용자가 자신의 친구들에게 의견을 빠르게 받도록 돕는 것

핵심 성공 조건:
👉 “배틀 생성 후 1시간 내 최소 5표 확보”

이 목표를 달성하지 못하는 기능은 구현하지 마라.

================================
🚫 금지 사항 (초기 MVP)
================================

아래 기능은 절대 구현하지 마라:

- 랜덤 매칭
- 공개 피드
- 글로벌 타임라인
- 팔로우/친구 시스템
- 랭킹 시스템
- 댓글
- 복잡한 AI 분석
- 로그인 강제 투표
- 앱 설치 강제 유도

이 앱은 초기에 “친구 초대 기반”으로만 작동해야 한다.

================================
🎯 핵심 제품 구조
================================

이 앱은 아래 흐름 하나만 완벽하게 만들면 된다:

1. 사용자가 사진 2장 업로드
2. 배틀 생성
3. 링크 생성
4. 친구에게 공유
5. 친구가 로그인 없이 바로 투표
6. 결과 확인
7. 일부 투표자가 새로운 생성자로 전환

이 흐름을 방해하는 요소는 모두 제거하라.

================================
3. 가장 중요한 기능 (우선순위 1)
================================

### 1. 배틀 생성 즉시 공유

사용자는:
- 사진 2장 선택
- 버튼 클릭
- 바로 공유 가능해야 한다

중간 단계 최소화

---

### 2. 로그인 없이 투표

절대 조건:
- 로그인 없이 투표 가능
- 앱 설치 없이 투표 가능
- 1탭 투표

Flow:
링크 클릭 → A vs B → 선택 → 끝

---

### 3. 투표 완료 후 전환 유도

투표 완료 화면에서:

- “지금까지 8명이 참여했어요”
- “나도 내 사진으로 의견 받아보기”

이 CTA는 자연스럽게 이어져야 한다.

---

### 4. 공유 메시지 자동 생성

공유 메시지는 매우 중요하다.

자동 생성 예시:

- “프로필 사진 뭐가 더 나은지 골라줘 🙏”
- “둘 중 하나만 찍어줘! 3초면 돼”
- “친구 의견 모으는 중인데 하나만 선택해줘”

AI Agent는 최소 5개 이상의 공유 메시지 템플릿을 구현하라.

================================
14. 통합 피드백 시스템 (AI + 사람)
================================

이제 Result 화면에 “통합 피드백 시스템”을 구현하라.

이 기능은 단순 결과 표시를 넘어
사용자가 “왜 이 사진이 더 나은지” 이해하도록 돕는 핵심 기능이다.

================================
🚨 절대 규칙 (중요)
================================

- 사람의 외모를 평가하지 마라
- attractiveness / beauty / 잘생김 / 예쁨 관련 표현 금지
- 사람 자체를 평가하지 말고 “사진”과 “인상”을 설명하라
- 모든 피드백은 중립적이고 도움을 주는 톤이어야 한다

================================
🎯 목표
================================

결과 화면에서 아래 3가지 제공:

1. 투표 결과
2. 사람 선택 이유 요약
3. AI 사진 피드백

그리고 이 3개를 합쳐서 “최종 인사이트”를 생성한다.

================================
1. 사람 피드백 시스템 (선택형)
================================

자유 텍스트 입력은 금지한다.
반드시 선택형 구조로 구현한다.

--------------------------------
투표 후 UX
--------------------------------

1. 사용자가 A/B 선택
2. 즉시 투표 완료
3. optional reason picker 표시
4. 최대 2개까지 선택 가능
5. 건너뛰기 가능

--------------------------------
허용 reason 목록
--------------------------------

- natural → 더 자연스러워 보여서
- bright_clear → 더 밝고 선명해서
- comfortable_expression → 표정이 더 편안해 보여서
- good_mood → 전체 분위기가 더 좋아서
- profile_fit → 프로필 사진으로 더 적합해서
- clean_impression → 더 깔끔한 느낌이어서
- friendly_impression → 더 친근한 인상이어서
- stable_composition → 사진 구도가 더 안정적이어서

금지:
- 잘생김 / 예쁨 / 매력 관련 reason

--------------------------------
DB 스키마
--------------------------------

vote_reasons
- id uuid pk
- vote_id uuid references votes(id)
- battle_id uuid references battles(id)
- selected_slot text ('A','B')
- reason_key text
- created_at timestamptz

--------------------------------
사람 피드백 요약 로직
--------------------------------

각 슬롯(A/B)에 대해:
- reason_key count 집계
- 상위 2개 reason 추출

문장 생성:

if 70% 이상:
→ “대부분의 사람들이 A 사진을 ‘자연스러움’, ‘프로필 적합도’ 이유로 선택했어요.”

if 55~70%:
→ “A 사진이 ‘자연스러움’ 측면에서 조금 더 선호되는 경향이 있어요.”

if 45~55%:
→ “두 사진 모두 비슷한 반응을 얻고 있어요.”

================================
2. AI 피드백 시스템
================================

AI는 “사진 품질”만 분석한다.

--------------------------------
분석 항목
--------------------------------

- brightness
- sharpness
- composition
- expression
- occlusion
- profile_suitability

--------------------------------
출력 형식
--------------------------------

interface AiFeedback {
  brightness: "good" | "ok" | "low"
  sharpness: "good" | "ok" | "low"
  composition: "good" | "ok" | "needs_improvement"
  expression: "natural" | "neutral" | "awkward"
  occlusion: boolean
  summary: string
}

--------------------------------
AI summary 예시
--------------------------------

- “밝기와 구도가 안정적이고 프로필용으로 무난한 사진이에요.”
- “표정이 자연스럽고 선명도가 좋아서 프로필 사진으로 적합해 보여요.”
- “조금 더 밝은 환경에서 촬영하면 더 좋은 결과가 나올 수 있어요.”

금지:
- 외모 평가

================================
3. 통합 인사이트 생성 (핵심)

--------------------------------
로직
--------------------------------

입력:
- vote ratio
- top reasons
- AI summary

출력:

예시:

- “투표 결과와 사진 피드백을 종합하면 A 사진이 더 자연스럽고 프로필용으로 적합하다는 의견이 많아요.”
- “두 사진 모두 비슷한 반응이지만 A는 자연스러운 느낌, B는 밝기 측면에서 장점이 있어요.”
- “B 사진은 밝고 선명하다는 평가가 있었지만 전체적으로 A가 조금 더 선호되는 경향이에요.”

--------------------------------
규칙
--------------------------------

- 사람 의견을 우선
- AI는 보조 역할
- 단정 금지 (“~해 보여요” 사용)
