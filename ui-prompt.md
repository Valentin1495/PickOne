이제 웹 투표 페이지 UI만 집중해서 다시 설계하고 구현해라.

이 화면은 제품에서 가장 중요한 전환 지점이다.
목표는 “클릭 → 투표 완료” 전환율을 극대화하는 것이다.

================================
핵심 UX 규칙 (절대 어기지 마라)
================================

- 로그인 없이 투표
- 앱 설치 요구 금지
- 스크롤 금지 (한 화면 완결)
- 버튼 금지 (“투표하기”, “확인” 등)
- 탭 = 즉시 투표
- 1초 내 행동 완료 가능해야 함

================================
레이아웃
================================

구조:

[상단]
- 질문 (짧고 자연스럽게)
- “3초면 끝 👇” 작은 문구

[중앙]
- A/B 이미지 2개
- 화면의 70~80% 차지
- 동일 크기
- 이미지 자체가 버튼

[하단]
- 없음 (스크롤 없음)

================================
인터랙션
================================

사용자 행동:

1. 이미지 탭
2. 선택된 이미지 강조 (scale up 또는 border)
3. 0.3초 후 결과 화면 이동

절대:
- confirm 버튼 만들지 마라

================================
투표 완료 화면
================================

구조:

- “투표 완료” 메시지
- 현재 투표 수 (예: 12명 참여)
- 간단 결과 (A 58% vs B 42%)

CTA:
- “나도 내 사진으로 의견 받아보기”

CTA는 강요하지 말고 자연스럽게 배치

================================
전환 최적화 요소
================================

반드시 포함:

- “익명으로 투표됩니다”
- 빠른 로딩 (skeleton 또는 placeholder)
- 이미지 먼저 렌더링
- 클릭 피드백 즉시

================================
금지 사항
================================

- 로그인 요구
- 회원가입 유도
- 광고
- 긴 텍스트
- 복잡한 UI

================================
출력 요구
================================

- Expo Web + React 기준 코드 작성
- 모바일 웹 최적화
- 컴포넌트 분리
- 스타일 포함
- 실제 동작 가능한 코드
